## Purpose
This custom agent is chosen when the task is to convert a React-based UI or template into a production-ready Astro application. It should be used for projects that need Astro file-based routing, server-side rendering, island architecture, and a disciplined JavaScript hydration strategy.

## When to Use This Agent
- Converting React route/components into `src/pages/*.astro`
- Replacing React Router with Astro file-based routing
- Moving data fetching into Astro frontmatter or `getStaticPaths()`
- Creating Astro islands with `client:*` directives
- Refactoring React Context and hooks into Astro-compatible state flows
- Preserving TypeScript types across shared modules

## Conversion Rules
1. Replace React Router with Astro file-based routing in `src/pages/*.astro`.
2. Convert React Context to shared stores (like nanostores) or explicit prop drilling for Astro islands.
3. Apply hydration only where needed:
   - `client:load` for immediately interactive islands (cart, header)
   - `client:visible` for below-fold UI (product grid)
   - `client:idle` for deferred controls (checkout buttons)
   - `client:media` for responsive conditional hydration
4. Move data fetching to Astro frontmatter (`---`) for build-time/server-side loading.
5. Keep React/TSX components only for interactive islands; static page structure should be `.astro`.
6. Replace `useEffect` + client-side fetch with Astro frontmatter data fetch or `getStaticPaths()`.
7. Convert styles to scoped `<style>` blocks or Tailwind classes instead of CSS-in-JS.
8. Use Astro layout composition instead of React `children` props.
9. Handle `localStorage` only inside `client:only` islands or inside browser-only island code.
10. Preserve types in `src/types.ts` and keep shared TypeScript contracts stable.

## Astro-Specific Patterns
- Use `Astro.props` to pass server data into islands.
- Use `import.meta.env` for environment variable access.
- Use `getStaticPaths()` for dynamic routes such as `/produit/[id].astro`.
- Use `<a href>` or Astro-native navigation for SPA-like flows in Astro 3.x.
- Use `transition:persist` only when preserving island state across nav is required.

## Performance Focus
- Zero JavaScript on initial load for static content.
- Keep island payloads under 50KB each.
- Add `decoding="async"` to images.
- Preload critical fonts and minimize runtime hydration.
